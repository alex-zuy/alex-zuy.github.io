62    // A note on memory visibility.
63    // Many of the utilities in this class (transform, withFallback, withTimeout, asList, combine)
64    // have two requirements that significantly complicate their design.
65    // 1. Cancellation should propagate from the returned future to the input future(s).
66    // 2. The returned futures shouldn't unnecessarily 'pin' their inputs after completion.
67    //
68    // A consequence of these these requirements is that the delegate futures cannot be stored in
69    // final fields.
70    //
71    // For simplicity the rest of this description will discuss Futures.withFallback since it is the
72    // simplest instance, though very similar descriptions apply to many other classes in this file.
73    //
74    // In the constructor of FutureFallback, the delegate future is assigned to a field 'running'.
75    // That field is non-final and non-volatile.  There are 2 places where the 'running' field is read
76    // and where we will have to consider visibility of the write operation in the constructor.
77    //
78    // 1. In the listener that performs the callback.  In this case it is fine since running is
79    //    assigned prior to calling addListener, and addListener happens-before any invocation of the
80    //    listener. Notably, this means that 'volatile' is unnecessary to make 'running' visible to
81    //    the listener.
82    //
83    // 2. In cancel() where we propagate cancellation to the input.  In this case it is _not_ fine.
84    //    There is currently nothing that enforces that the write to running in the constructor is
85    //    visible to cancel().  This is because there is no happens before edge between the write and
86    //    a (hypothetical) unsafe read by our caller. Note: adding 'volatile' does not fix this issue,
87    //    it would just add an edge such that if cancel() observed non-null, then it would also
88    //    definitely observe all earlier writes, but we still have no guarantee that cancel() would
89    //    see the inital write (just stronger guarantees if it does).
90    //
91    // See: http://cs.oswego.edu/pipermail/concurrency-interest/2015-January/013800.html
92    // For a (long) discussion about this specific issue and the general futility of life.
93    //
94    // For the time being we are OK with the problem discussed above since it requires a caller to
95    // introduce a very specific kind of data-race.  And given the other operations performed by these
96    // methods that involve volatile read/write operations, in practice there is no issue.  Also, the
97    // way in such a visibility issue would surface is most likely as a failure of cancel() to
98    // propagate to the input.  Cancellation propagation is fundamentally racy so this is fine.
99    //
100   // Future versions of the JMM may revise safe construction semantics in such a way that we can
101   // safely publish these objects and we won't need this whole discussion.
102   // TODO(user,lukes): consider adding volatile to all these fields since in current known JVMs
103   // that should resolve the issue.  This comes at the cost of adding more write barriers to the
104   // implementations.

108   private abstract static class ImmediateFuture<V>
109       implements ListenableFuture<V> {
110 
111     private static final Logger log =
112         Logger.getLogger(ImmediateFuture.class.getName());
113 
114     @Override
115     public void addListener(Runnable listener, Executor executor) {
116       checkNotNull(listener, "Runnable was null.");
117       checkNotNull(executor, "Executor was null.");
118       try {
119         executor.execute(listener);
120       } catch (RuntimeException e) {
121         // ListenableFuture's contract is that it will not throw unchecked
122         // exceptions, so log the bad runnable and/or executor and swallow it.
123         log.log(Level.SEVERE, "RuntimeException while executing runnable "
124             + listener + " with executor " + executor, e);
125       }
126     }
127 
128     @Override
129     public boolean cancel(boolean mayInterruptIfRunning) {
130       return false;
131     }
132 
133     @Override
134     public abstract V get() throws ExecutionException;
135 
136     @Override
137     public V get(long timeout, TimeUnit unit) throws ExecutionException {
138       checkNotNull(unit);
139       return get();
140     }
141 
142     @Override
143     public boolean isCancelled() {
144       return false;
145     }
146 
147     @Override
148     public boolean isDone() {
149       return true;
150     }
151   }

178   /**
179    * Future that delegates to another but will finish early (via a {@link
180    * TimeoutException} wrapped in an {@link ExecutionException}) if the
181    * specified duration expires.
182    * The delegate future is interrupted and cancelled if it times out.
183    */
184   private static final class TimeoutFuture<V> extends AbstractFuture.TrustedFuture<V> {
185     // Memory visibility of these fields.
186     // There are two cases to consider.
187     // 1. visibility of the writes to these fields to Fire.run
188     //    The initial write to delegateRef is made definitely visible via the semantics of
189     //    addListener/SES.schedule.  The later racy write in cancel() is not guaranteed to be
190     //    observed, however that is fine since the correctness is based on the atomic state in
191     //    our base class.
192     //    The initial write to timer is never definitely visible to Fire.run since it is assigned
193     //    after SES.schedule is called. Therefore Fire.run has to check for null.  However, it
194     //    should be visible if Fire.run is called by delegate.addListener since addListener is
195     //    called after the assignment to timer, and importantly this is the main situation in which
196     //    we need to be able to see the write.
197     // 2. visibility of the writes to cancel
198     //    Since these fields are non-final that means that TimeoutFuture is not being 'safely
199     //    published', thus a motivated caller may be able to expose the reference to another thread
200     //    that would then call cancel() and be unable to cancel the delegate.
201     //    There are a number of ways to solve this, none of which are very pretty, and it is
202     //    currently believed to be a purely theoretical problem (since the other actions should
203     //    supply sufficient write-barriers).
204 
205     ListenableFuture<V> delegateRef;
206     Future<?> timer;
207 
208     TimeoutFuture(ListenableFuture<V> delegate) {
209       this.delegateRef = Preconditions.checkNotNull(delegate);
210     }
211 
212     /** A runnable that is called when the delegate or the timer completes. */
213     private static final class Fire<V> implements Runnable {
214       // Holding a strong reference to the enclosing class (as we would do if
215       // this weren't a static nested class) could cause retention of the
216       // delegate's return value (in AbstractFuture) for the duration of the
217       // timeout in the case of successful completion. We clear this on run.
218       TimeoutFuture<V> timeoutFutureRef;
219 
220       Fire(TimeoutFuture<V> timeoutFuture) {
221         this.timeoutFutureRef = timeoutFuture;
222       }
223 
224       @Override public void run() {
225         // If either of these reads return null then we must be after a successful cancel
226         // or another call to this method.
227         TimeoutFuture<V> timeoutFuture = timeoutFutureRef;
228         if (timeoutFuture == null) {
229           return;
230         }
231         ListenableFuture<V> delegate = timeoutFuture.delegateRef;
232         if (delegate == null) {
233           return;
234         }
235 
236         // Unpin all the memory before attempting to complete.  Not only does this save us from
237         // wrapping everything in a finally block, it also ensures that if delegate.cancel() (in the
238         // else block), causes delegate to complete, then it won't reentrantly call back in and
239         // cause TimeoutFuture to finish with cancellation.
240         timeoutFutureRef = null;
241         Future<?> timer = timeoutFuture.timer;
242         timeoutFuture.delegateRef = null;
243         timeoutFuture.timer = null;
244         if (delegate.isDone()) {
245           timeoutFuture.setFuture(delegate);
246           // Try to cancel the timer as an optimization
247           // timer may be null if this call to run was by the timer task since there is no
248           // happens-before edge between the assignment to timer and an execution of the timer
249           // task.
250           if (timer != null) {
251             timer.cancel(false);
252           }
253         } else {
254           // Some users, for better or worse, rely on the delegate definitely being cancelled prior
255           // to the timeout future completing.  We wrap in a try...finally... for the off chance
256           // that cancelling the delegate causes an Error to be thrown from a listener on the
257           // delegate.
258           try {
259             delegate.cancel(true);
260           } finally {
261             // TODO(lukes): this stack trace is particularly useless (all it does is point at the
262             // scheduledexecutorservice thread), consider eliminating it altogether?
263             timeoutFuture.setException(new TimeoutException("Future timed out: " + delegate));
264           }
265         }
266       }
267     }
268 
269     @Override public boolean cancel(boolean mayInterruptIfRunning) {
270       Future<?> localTimer = timer;
271       ListenableFuture<V> delegate = delegateRef;
272       if (super.cancel(mayInterruptIfRunning)) {
273         // Either can be null if super.cancel() races with an execution of Fire.run, but it doesn't
274         // matter because either 1. the delegate is already done (so there is no point in
275         // propagating cancellation and Fire.run will cancel the timer. or 2. the timeout occurred
276         // and Fire.run will cancel the delegate
277         // Technically this is also possible in the 'unsafe publishing' case described above.
278         if (delegate != null) {
279           // Unpin and prevent Fire from doing anything if delegate.cancel finishes the delegate.
280           delegateRef = null;
281           delegate.cancel(mayInterruptIfRunning);
282         }
283         if (localTimer != null) {
284           timer = null;
285           localTimer.cancel(false);
286         }
287         return true;
288       }
289       return false;
290     }
291   }

293   /**
294    * Returns a new {@code ListenableFuture} whose result is asynchronously
295    * derived from the result of the given {@code Future}. More precisely, the
296    * returned {@code Future} takes its result from a {@code Future} produced by
297    * applying the given {@code AsyncFunction} to the result of the original
298    * {@code Future}. Example:
299    *
300    * <pre>   {@code
301    *   ListenableFuture<RowKey> rowKeyFuture = indexService.lookUp(query);
302    *   AsyncFunction<RowKey, QueryResult> queryFunction =
303    *       new AsyncFunction<RowKey, QueryResult>() {
304    *         public ListenableFuture<QueryResult> apply(RowKey rowKey) {
305    *           return dataService.read(rowKey);
306    *         }
307    *       };
308    *   ListenableFuture<QueryResult> queryFuture =
309    *       transform(rowKeyFuture, queryFunction);}</pre>
310    *
311    * <p>Note: If the derived {@code Future} is slow or heavyweight to create
312    * (whether the {@code Future} itself is slow or heavyweight to complete is
313    * irrelevant), consider {@linkplain #transform(ListenableFuture,
314    * AsyncFunction, Executor) supplying an executor}. If you do not supply an
315    * executor, {@code transform} will use a
316    * {@linkplain MoreExecutors#directExecutor direct executor}, which carries
317    * some caveats for heavier operations. For example, the call to {@code
318    * function.apply} may run on an unpredictable or undesirable thread:
319    *
320    * <ul>
321    * <li>If the input {@code Future} is done at the time {@code transform} is
322    * called, {@code transform} will call {@code function.apply} inline.
323    * <li>If the input {@code Future} is not yet done, {@code transform} will
324    * schedule {@code function.apply} to be run by the thread that completes the
325    * input {@code Future}, which may be an internal system thread such as an
326    * RPC network thread.
327    * </ul>
328    *
329    * <p>Also note that, regardless of which thread executes the {@code
330    * function.apply}, all other registered but unexecuted listeners are
331    * prevented from running during its execution, even if those listeners are
332    * to run in other executors.
333    *
334    * <p>The returned {@code Future} attempts to keep its cancellation state in
335    * sync with that of the input future and that of the future returned by the
336    * function. That is, if the returned {@code Future} is cancelled, it will
337    * attempt to cancel the other two, and if either of the other two is
338    * cancelled, the returned {@code Future} will receive a callback in which it
339    * will attempt to cancel itself.
340    *
341    * @param input The future to transform
342    * @param function A function to transform the result of the input future
343    *     to the result of the output future
344    * @return A future that holds result of the function (if the input succeeded)
345    *     or the original input's failure (if not)
346    * @since 11.0
347    */
348   public static <I, O> ListenableFuture<O> transform(ListenableFuture<I> input,
349       AsyncFunction<? super I, ? extends O> function) {
350     ChainingListenableFuture<I, O> output =
351         new ChainingListenableFuture<I, O>(function, input);
352     input.addListener(output, directExecutor());
353     return output;
354   }

356   /**
357    * Returns a new {@code ListenableFuture} whose result is asynchronously
358    * derived from the result of the given {@code Future}. More precisely, the
359    * returned {@code Future} takes its result from a {@code Future} produced by
360    * applying the given {@code AsyncFunction} to the result of the original
361    * {@code Future}. Example:
362    *
363    * <pre>   {@code
364    *   ListenableFuture<RowKey> rowKeyFuture = indexService.lookUp(query);
365    *   AsyncFunction<RowKey, QueryResult> queryFunction =
366    *       new AsyncFunction<RowKey, QueryResult>() {
367    *         public ListenableFuture<QueryResult> apply(RowKey rowKey) {
368    *           return dataService.read(rowKey);
369    *         }
370    *       };
371    *   ListenableFuture<QueryResult> queryFuture =
372    *       transform(rowKeyFuture, queryFunction, executor);}</pre>
373    *
374    * <p>The returned {@code Future} attempts to keep its cancellation state in
375    * sync with that of the input future and that of the future returned by the
376    * chain function. That is, if the returned {@code Future} is cancelled, it
377    * will attempt to cancel the other two, and if either of the other two is
378    * cancelled, the returned {@code Future} will receive a callback in which it
379    * will attempt to cancel itself.
380    *
381    * <p>When the execution of {@code function.apply} is fast and lightweight
382    * (though the {@code Future} it returns need not meet these criteria),
383    * consider {@linkplain #transform(ListenableFuture, AsyncFunction) omitting
384    * the executor} or explicitly specifying {@code directExecutor}.
385    * However, be aware of the caveats documented in the link above.
386    *
387    * @param input The future to transform
388    * @param function A function to transform the result of the input future
389    *     to the result of the output future
390    * @param executor Executor to run the function in.
391    * @return A future that holds result of the function (if the input succeeded)
392    *     or the original input's failure (if not)
393    * @since 11.0
394    */
395   public static <I, O> ListenableFuture<O> transform(ListenableFuture<I> input,
396       AsyncFunction<? super I, ? extends O> function,
397       Executor executor) {
398     checkNotNull(executor);
399     ChainingListenableFuture<I, O> output =
400         new ChainingListenableFuture<I, O>(function, input);
401     input.addListener(rejectionPropagatingRunnable(output, output, executor), directExecutor());
402     return output;
403   }

405   /**
406    * Returns a Runnable that will invoke the delegate Runnable on the delegate executor, but if the
407    * task is rejected, it will propagate that rejection to the output future.
408    */
409   private static Runnable rejectionPropagatingRunnable(
410       final AbstractFuture<?> outputFuture,
411       final Runnable delegateTask,
412       final Executor delegateExecutor) {
413     return new Runnable() {
414       @Override public void run() {
415         final AtomicBoolean thrownFromDelegate = new AtomicBoolean(true);
416         try {
417           delegateExecutor.execute(new Runnable() {
418             @Override public void run() {
419               thrownFromDelegate.set(false);
420               delegateTask.run();
421             }
422           });
423         } catch (RejectedExecutionException e) {
424           if (thrownFromDelegate.get()) {
425             // wrap exception?
426             outputFuture.setException(e);
427           }
428           // otherwise it must have been thrown from a transitive call and the delegate runnable
429           // should have handled it.
430         }
431       }
432     };
433   }

435   /**
436    * Returns a new {@code ListenableFuture} whose result is the product of
437    * applying the given {@code Function} to the result of the given {@code
438    * Future}. Example:
439    *
440    * <pre>   {@code
441    *   ListenableFuture<QueryResult> queryFuture = ...;
442    *   Function<QueryResult, List<Row>> rowsFunction =
443    *       new Function<QueryResult, List<Row>>() {
444    *         public List<Row> apply(QueryResult queryResult) {
445    *           return queryResult.getRows();
446    *         }
447    *       };
448    *   ListenableFuture<List<Row>> rowsFuture =
449    *       transform(queryFuture, rowsFunction);}</pre>
450    *
451    * <p>Note: If the transformation is slow or heavyweight, consider {@linkplain
452    * #transform(ListenableFuture, Function, Executor) supplying an executor}.
453    * If you do not supply an executor, {@code transform} will use an inline
454    * executor, which carries some caveats for heavier operations.  For example,
455    * the call to {@code function.apply} may run on an unpredictable or
456    * undesirable thread:
457    *
458    * <ul>
459    * <li>If the input {@code Future} is done at the time {@code transform} is
460    * called, {@code transform} will call {@code function.apply} inline.
461    * <li>If the input {@code Future} is not yet done, {@code transform} will
462    * schedule {@code function.apply} to be run by the thread that completes the
463    * input {@code Future}, which may be an internal system thread such as an
464    * RPC network thread.
465    * </ul>
466    *
467    * <p>Also note that, regardless of which thread executes the {@code
468    * function.apply}, all other registered but unexecuted listeners are
469    * prevented from running during its execution, even if those listeners are
470    * to run in other executors.
471    *
472    * <p>The returned {@code Future} attempts to keep its cancellation state in
473    * sync with that of the input future. That is, if the returned {@code Future}
474    * is cancelled, it will attempt to cancel the input, and if the input is
475    * cancelled, the returned {@code Future} will receive a callback in which it
476    * will attempt to cancel itself.
477    *
478    * <p>An example use of this method is to convert a serializable object
479    * returned from an RPC into a POJO.
480    *
481    * @param input The future to transform
482    * @param function A Function to transform the results of the provided future
483    *     to the results of the returned future.  This will be run in the thread
484    *     that notifies input it is complete.
485    * @return A future that holds result of the transformation.
486    * @since 9.0 (in 1.0 as {@code compose})
487    */
488   public static <I, O> ListenableFuture<O> transform(ListenableFuture<I> input,
489       final Function<? super I, ? extends O> function) {
490     checkNotNull(function);
491     ChainingListenableFuture<I, O> output =
492         new ChainingListenableFuture<I, O>(asAsyncFunction(function), input);
493     input.addListener(output, directExecutor());
494     return output;
495   }

540   /** Wraps the given function as an AsyncFunction. */

551   /**
552    * An implementation of {@code ListenableFuture} that also implements
553    * {@code Runnable} so that it can be used to nest ListenableFutures.
554    * Once the passed-in {@code ListenableFuture} is complete, it calls the
555    * passed-in {@code Function} to generate the result.
556    *
557    * <p>For historical reasons, this class has a special case in its exception
558    * handling: If the given {@code AsyncFunction} throws an {@code
559    * UndeclaredThrowableException}, {@code ChainingListenableFuture} unwraps it
560    * and uses its <i>cause</i> as the output future's exception, rather than
561    * using the {@code UndeclaredThrowableException} itself as it would for other
562    * exception types. The reason for this is that {@code Futures.transform} used
563    * to require a {@code Function}, whose {@code apply} method is not allowed to
564    * throw checked exceptions. Nowadays, {@code Futures.transform} has an
565    * overload that accepts an {@code AsyncFunction}, whose {@code apply} method
566    * <i>is</i> allowed to throw checked exception. Users who wish to throw
567    * checked exceptions should use that overload instead, and <a
568    * href="http://code.google.com/p/guava-libraries/issues/detail?id=1548">we
569    * should remove the {@code UndeclaredThrowableException} special case</a>.
570    */
571   private static final class ChainingListenableFuture<I, O>
572       extends AbstractFuture.TrustedFuture<O> implements Runnable {
573 
574     private AsyncFunction<? super I, ? extends O> function;
575     // In theory, this field might not be visible to a cancel() call in certain circumstances. For
576     // details, see the comments on the fields of TimeoutFuture.
577     private ListenableFuture<? extends I> inputFuture;
578 
579     private ChainingListenableFuture(
580         AsyncFunction<? super I, ? extends O> function,
581         ListenableFuture<? extends I> inputFuture) {
582       this.function = checkNotNull(function);
583       this.inputFuture = checkNotNull(inputFuture);
584     }
585 
586     @Override
587     public boolean cancel(boolean mayInterruptIfRunning) {
588       /*
589        * Our additional cancellation work needs to occur even if
590        * !mayInterruptIfRunning, so we can't move it into interruptTask().
591        */
592       if (super.cancel(mayInterruptIfRunning)) {
593         ListenableFuture<? extends I> localInputFuture = inputFuture;
594         if (localInputFuture != null) {
595           localInputFuture.cancel(mayInterruptIfRunning);
596         }
597         return true;
598       }
599       return false;
600     }
601 
602     @Override
603     public void run() {
604       try {
605         I sourceResult;
606         try {
607           sourceResult = getUninterruptibly(inputFuture);
608         } catch (CancellationException e) {
609           // Cancel this future and return.
610           // At this point, inputFuture is cancelled and outputFuture doesn't
611           // exist, so the value of mayInterruptIfRunning is irrelevant.
612           cancel(false);
613           return;
614         } catch (ExecutionException e) {
615           // Set the cause of the exception as this future's exception
616           setException(e.getCause());
617           return;
618         }
619 
620         ListenableFuture<? extends O> outputFuture = function.apply(sourceResult);
621         checkNotNull(outputFuture, "AsyncFunction.apply returned null instead of a Future. "
622             + "Did you mean to return immediateFuture(null)?");
623         setFuture(outputFuture);
624       } catch (UndeclaredThrowableException e) {
625         // Set the cause of the exception as this future's exception
626         setException(e.getCause());
627       } catch (Throwable t) {
628         // This exception is irrelevant in this thread, but useful for the
629         // client
630         setException(t);
631       } finally {
632         // Don't pin inputs beyond completion
633         function = null;
634         inputFuture = null;
635       }
636     }
637   }

639   /**
640    * Returns a new {@code ListenableFuture} whose result is the product of
641    * calling {@code get()} on the {@code Future} nested within the given {@code
642    * Future}, effectively chaining the futures one after the other.  Example:
643    *
644    * <pre>   {@code
645    *   SettableFuture<ListenableFuture<String>> nested = SettableFuture.create();
646    *   ListenableFuture<String> dereferenced = dereference(nested);}</pre>
647    *
648    * <p>This call has the same cancellation and execution semantics as {@link
649    * #transform(ListenableFuture, AsyncFunction)}, in that the returned {@code
650    * Future} attempts to keep its cancellation state in sync with both the
651    * input {@code Future} and the nested {@code Future}.  The transformation
652    * is very lightweight and therefore takes place in the same thread (either
653    * the thread that called {@code dereference}, or the thread in which the
654    * dereferenced future completes).
655    *
656    * @param nested The nested future to transform.
657    * @return A future that holds result of the inner future.
658    * @since 13.0
659    */
660   @SuppressWarnings({"rawtypes", "unchecked"})
661   public static <V> ListenableFuture<V> dereference(
662       ListenableFuture<? extends ListenableFuture<? extends V>> nested) {
663     return Futures.transform((ListenableFuture) nested, (AsyncFunction) DEREFERENCER);
664   }

667    * Helper {@code Function} for {@link #dereference}.
668    */
669   private static final AsyncFunction<ListenableFuture<Object>, Object> DEREFERENCER =
670       new AsyncFunction<ListenableFuture<Object>, Object>() {
671         @Override public ListenableFuture<Object> apply(ListenableFuture<Object> input) {
672           return input;
673         }

676   private static final class CombinedFuture<V> extends TrustedListenableFutureTask<V> {
677     ImmutableList<ListenableFuture<?>> futures;
678 
679     CombinedFuture(Callable<V> callable, ImmutableList<ListenableFuture<?>> futures) {
680       super(callable);
681       this.futures = futures;
682     }
683 
684     @Override void doRun(Callable<V> task) throws Exception {
685       // Very similar to the default implementation, but has specialized handling of
686       // ExecutionExceptions and CancellationExceptions
687       try {
688         set(task.call());
689       } catch (ExecutionException e) {
690         setException(e.getCause());
691       } catch (CancellationException e) {
692         cancel(false);
693       }
694     }
695 
696     @Override public boolean cancel(boolean mayInterruptIfRunning) {
697       ImmutableList<ListenableFuture<?>> localFutures = this.futures;
698       if (super.cancel(mayInterruptIfRunning)) {
699         if (localFutures != null) {
700           for (ListenableFuture<?> future : localFutures) {
701             future.cancel(mayInterruptIfRunning);
702           }
703         }
704         return true;
705       }
706       return false;
707     }
708 
709     @Override void done() {
710       futures = null;
711     }
712   }

714   /**
715    * Registers separate success and failure callbacks to be run when the {@code
716    * Future}'s computation is {@linkplain java.util.concurrent.Future#isDone()
717    * complete} or, if the computation is already complete, immediately.
718    *
719    * <p>There is no guaranteed ordering of execution of callbacks, but any
720    * callback added through this method is guaranteed to be called once the
721    * computation is complete.
722    *
723    * Example: <pre> {@code
724    * ListenableFuture<QueryResult> future = ...;
725    * addCallback(future,
726    *     new FutureCallback<QueryResult> {
727    *       public void onSuccess(QueryResult result) {
728    *         storeInCache(result);
729    *       }
730    *       public void onFailure(Throwable t) {
731    *         reportError(t);
732    *       }
733    *     });}</pre>
734    *
735    * <p>Note: If the callback is slow or heavyweight, consider {@linkplain
736    * #addCallback(ListenableFuture, FutureCallback, Executor) supplying an
737    * executor}. If you do not supply an executor, {@code addCallback} will use
738    * a {@linkplain MoreExecutors#directExecutor direct executor}, which carries
739    * some caveats for heavier operations. For example, the callback may run on
740    * an unpredictable or undesirable thread:
741    *
742    * <ul>
743    * <li>If the input {@code Future} is done at the time {@code addCallback} is
744    * called, {@code addCallback} will execute the callback inline.
745    * <li>If the input {@code Future} is not yet done, {@code addCallback} will
746    * schedule the callback to be run by the thread that completes the input
747    * {@code Future}, which may be an internal system thread such as an RPC
748    * network thread.
749    * </ul>
750    *
751    * <p>Also note that, regardless of which thread executes the callback, all
752    * other registered but unexecuted listeners are prevented from running
753    * during its execution, even if those listeners are to run in other
754    * executors.
755    *
756    * <p>For a more general interface to attach a completion listener to a
757    * {@code Future}, see {@link ListenableFuture#addListener addListener}.
758    *
759    * @param future The future attach the callback to.
760    * @param callback The callback to invoke when {@code future} is completed.
761    * @since 10.0
762    */
763   public static <V> void addCallback(ListenableFuture<V> future,
764       FutureCallback<? super V> callback) {
765     addCallback(future, callback, directExecutor());
766   }

768   /**
769    * Registers separate success and failure callbacks to be run when the {@code
770    * Future}'s computation is {@linkplain java.util.concurrent.Future#isDone()
771    * complete} or, if the computation is already complete, immediately.
772    *
773    * <p>The callback is run in {@code executor}.
774    * There is no guaranteed ordering of execution of callbacks, but any
775    * callback added through this method is guaranteed to be called once the
776    * computation is complete.
777    *
778    * Example: <pre> {@code
779    * ListenableFuture<QueryResult> future = ...;
780    * Executor e = ...
781    * addCallback(future,
782    *     new FutureCallback<QueryResult> {
783    *       public void onSuccess(QueryResult result) {
784    *         storeInCache(result);
785    *       }
786    *       public void onFailure(Throwable t) {
787    *         reportError(t);
788    *       }
789    *     }, e);}</pre>
790    *
791    * <p>When the callback is fast and lightweight, consider {@linkplain
792    * #addCallback(ListenableFuture, FutureCallback) omitting the executor} or
793    * explicitly specifying {@code directExecutor}. However, be aware of the
794    * caveats documented in the link above.
795    *
796    * <p>For a more general interface to attach a completion listener to a
797    * {@code Future}, see {@link ListenableFuture#addListener addListener}.
798    *
799    * @param future The future attach the callback to.
800    * @param callback The callback to invoke when {@code future} is completed.
801    * @param executor The executor to run {@code callback} when the future
802    *    completes.
803    * @since 10.0
804    */
805   public static <V> void addCallback(final ListenableFuture<V> future,
806       final FutureCallback<? super V> callback, Executor executor) {
807     Preconditions.checkNotNull(callback);
808     Runnable callbackListener = new Runnable() {
809       @Override
810       public void run() {
811         final V value;
812         try {
813           // TODO(user): (Before Guava release), validate that this
814           // is the thing for IE.
815           value = getUninterruptibly(future);
816         } catch (ExecutionException e) {
817           callback.onFailure(e.getCause());
818           return;
819         } catch (RuntimeException e) {
820           callback.onFailure(e);
821           return;
822         } catch (Error e) {
823           callback.onFailure(e);
824           return;
825         }
826         callback.onSuccess(value);
827       }
828     };
829     future.addListener(callbackListener, executor);
830   }
